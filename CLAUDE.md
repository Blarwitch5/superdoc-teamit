# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project context

`teamit-superdoc` is a new, empty documentation site for Team IT (Swiss IT solutions). It is the successor to the sibling project `teamit-doc` (`../teamit-doc`), which is built with **Astro + Starlight + Tailwind CSS**. Expect this project to follow the same stack unless explicitly stated otherwise.

Reference the sibling projects for conventions:
- `../teamit-doc` — existing Astro + Starlight doc site (structure, content organisation, i18n pattern)
- `../teamit` — main marketing site (component conventions, TypeScript style)

## Expected stack

- **Astro** with `@astrojs/starlight` for docs framework
- **Tailwind CSS** via `@astrojs/starlight-tailwind` (`applyBaseStyles: false`)
- **MDX** for rich doc pages
- **pnpm** as package manager
- Bilingual: French (root locale) + English (`/en/` prefix)

## Commands (once initialised)

```bash
pnpm dev        # dev server at localhost:4321
pnpm build      # astro check && astro build
pnpm preview    # preview production build
```

## Code conventions (from sibling projects)

- TypeScript for all `.ts`/`.astro` files; prefer `type` over `interface`; no enums
- Lowercase-with-dashes for directory names (e.g. `components/auth-wizard`)
- Each component lives in its own folder with two files:
  - `index.ts` — re-export: `export { default as Hero } from "./Hero.astro"`
  - `ComponentName.astro` — the component itself
- Named exports preferred; functional, declarative style; no classes
- Descriptive variable names with auxiliary verbs (`isLoading`, `hasError`)

## Content structure (Starlight pattern)

Docs live in `src/content/docs/`. Each directory under `docs/` maps to a sidebar section autogenerated in `astro.config.mjs`. Translations mirror the same path under `src/content/docs/en/`.

## i18n

Default locale is French (`root`). English translations go in `src/content/docs/en/`. UI strings go in `src/content/i18n/fr.json` and `src/content/i18n/en.json`.

## Design tokens & couleurs

### Primitives de marque (src/styles/global.css)

Dérivées du logo officiel Team IT :

| Token | Valeur | Usage |
|-------|--------|-------|
| `--ti-red` | `#df3123` | Rouge logo — accent principal, boutons, badges |
| `--ti-red-hover` | `#c42b1e` | Rouge foncé — état hover |
| `--ti-red-bright` | `#e85748` | Rouge plus clair — liens/textes sur fond dark |
| `--ti-dark` | `#1b1b18` | Gris foncé logo — bg dark theme, texte light theme |
| `--ti-white` | `#ffffff` | Blanc pur |

Ces primitives sont disponibles **globalement** (Starlight + AppLayout).  
Ne jamais hardcoder `#df3123`, `#1b1b18`, `#ffffff` — utiliser les tokens.

### Règles

- Toujours utiliser `var(--token)` — jamais de couleur hex directe dans les styles.
- `--danger` / `--danger-bg` pour tous les états d'erreur (pas `#f87171`).
- Le logo Team IT nécessite un fond blanc fixe (`var(--ti-white)`) quelle que soit le thème.
- La page login force `data-theme="dark"` via prop `forceDark` sur `AppLayout` — ne pas modifier.
- Sur Starlight, utiliser les variables `--sl-color-*` (déjà overridées dans global.css).
