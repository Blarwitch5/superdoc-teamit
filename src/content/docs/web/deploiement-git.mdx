## Contexte

Le déploiement Git permet de mettre à jour automatiquement le site sur le serveur à chaque `git push`, sans passer par FTP. Infomaniak supporte cette méthode via son interface Manager.

## Prérequis

- Hébergement Infomaniak actif
- Dépôt Git existant (GitHub, GitLab, Bitbucket ou dépôt Infomaniak)
- Accès SSH ou accès à l'interface Manager

## Configuration via le Manager

1. Se connecter à [manager.infomaniak.com](https://manager.infomaniak.com).
2. Aller dans **Hébergement Web → [nom du service] → Déploiement Git**.
3. Cliquer sur **Activer le déploiement Git**.
4. Renseigner :
   - **URL du dépôt** : URL HTTPS ou SSH du dépôt (ex. : `https://github.com/organisation/monsite.git`)
   - **Branche** : la branche à déployer (ex. : `main` ou `production`)
   - **Dossier de destination** : généralement `/web/`
5. Si dépôt privé, ajouter la **clé SSH publique** générée par Infomaniak dans les clés autorisées du dépôt (GitHub : Settings → Deploy keys).
6. Cliquer sur **Enregistrer**.

## Déploiement manuel depuis Manager

Après configuration, cliquer sur **Déployer** dans l'interface pour déclencher un déploiement immédiat.

## Déploiement automatique via webhook

Pour déclencher un déploiement à chaque `git push` :

1. Copier l'**URL du webhook** affichée dans le Manager.
2. Sur GitHub : **Settings → Webhooks → Add webhook**.
   - Payload URL : l'URL copiée
   - Content type : `application/json`
   - Événement : `push`
3. Cliquer sur **Add webhook**.

Désormais, chaque `git push` sur la branche configurée déclenche automatiquement un déploiement.

## Déploiement via SSH (avancé)

Si l'hébergement dispose d'un accès SSH :

```bash
# Depuis le serveur, dans le dossier du site
git pull origin main
```

Ou via un script de déploiement déclenché par webhook/CI.

## Bonnes pratiques

- Ne jamais déployer directement sur `main` — travailler sur une branche de développement et merger après validation.
- Configurer un environnement de **staging** sur une branche dédiée avant la production.
- Vérifier que le fichier `.gitignore` exclut les fichiers sensibles (`.env`, `vendor/`, `node_modules/`).
- Tester le déploiement sur un site de test avant de configurer la production.
