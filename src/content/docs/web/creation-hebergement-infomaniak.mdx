## Accès

1. Se connecter à [manager.infomaniak.com](https://manager.infomaniak.com).
2. Dans le menu latéral, aller dans **Hébergement Web**.

## Création d'un hébergement

1. Cliquer sur **Commander un hébergement** (si aucun hébergement existant) ou **Ajouter un site** sur un hébergement mutualisé existant.
2. Choisir le **type d'hébergement** adapté :
   - **Mutualisé** : pour sites vitrines et PME
   - **Cloud Server** : pour applications nécessitant plus de ressources
3. Sélectionner le **plan** selon les besoins (espace disque, trafic, fonctionnalités).

## Configuration du domaine

1. Associer un domaine existant ou en enregistrer un nouveau.
2. Si le domaine est externe (autre registrar), modifier les **serveurs DNS** pour pointer vers Infomaniak :
   - `ns1.infomaniak.com`
   - `ns2.infomaniak.com`
3. La propagation DNS prend de **15 minutes à 48 heures**.

## Configuration initiale après création

### FTP

1. Aller dans **FTP → Créer un utilisateur FTP** (ou noter l'utilisateur par défaut).
2. Conserver les identifiants pour le déploiement des fichiers (voir [Accès FTP/SFTP](/serveur/acces-ftp-sftp/)).

### Base de données

Si l'application nécessite une base de données, la créer dès maintenant (voir [Création d'une base de données](/serveur/creation-base-de-donnees/)).

### SSL

Activer le certificat SSL Let's Encrypt immédiatement après la création (voir [Certificat SSL](/web/certificat-ssl/)).

### PHP

Aller dans **PHP → Version** et sélectionner la version requise par l'application (vérifier les prérequis du CMS ou du framework).

## Déploiement du site

- **Méthode FTP** : uploader les fichiers via FileZilla dans le dossier `/web/`.
- **Méthode Git** : configurer un déploiement automatique (voir [Déploiement Git](/web/deploiement-git/)).
- **Installation CMS** : utiliser l'installateur 1-clic Infomaniak (WordPress, Joomla, Prestashop…).

## Vérification

Une fois les fichiers déployés, tester le site en HTTPS :
