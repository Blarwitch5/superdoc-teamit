## Prérequis

- **FileZilla** installé ([filezilla-project.org](https://filezilla-project.org/))
- Identifiants FTP disponibles dans Infomaniak Manager

## Récupérer les identifiants FTP

1. Se connecter à [manager.infomaniak.com](https://manager.infomaniak.com).
2. Aller dans **Hébergement Web → [nom du service] → FTP**.
3. Copier :
   - **Hôte** (ex. : `ftp.cluster0XX.hosting.ovh.net` ou `ftp.infomaniak.com`)
   - **Nom d'utilisateur**
   - **Port** : `21` (FTP) ou `22` (SFTP)

Le mot de passe FTP peut être celui du compte ou un mot de passe dédié à réinitialiser depuis cette interface.

## Configuration dans FileZilla

### Via le Gestionnaire de sites (recommandé)

1. Ouvrir FileZilla → **Fichier → Gestionnaire de sites → Nouveau site**.
2. Renseigner :
   - **Protocole** : `SFTP - SSH File Transfer Protocol` (préférer SFTP pour la sécurité) ou `FTP`
   - **Hôte** : l'hôte copié depuis Manager
   - **Port** : `22` pour SFTP, `21` pour FTP
   - **Mode de connexion** : Normale
   - **Identifiant / Mot de passe** : identifiants FTP
3. Cliquer sur **Connexion**.

## Navigation et transfert

- Le panneau gauche affiche les fichiers locaux, le droit les fichiers distants.
- La racine du site web est généralement dans `/web/` ou `/htdocs/`.
- Pour transférer : glisser-déposer entre les deux panneaux, ou clic droit → **Envoyer**.

## Bonnes pratiques

- Utiliser **SFTP** plutôt que FTP (connexion chiffrée).
- Ne pas modifier les fichiers directement sur le serveur — travailler en local et uploader.
- Sauvegarder avant toute modification importante (voir [Sauvegarde de site](/serveur/sauvegarde-site/)).

## Dépannage

- **Connexion refusée** : vérifier le port (21 vs 22) et les identifiants.
- **Timeout** : vérifier que le pare-feu ne bloque pas les ports FTP/SFTP.
- **Erreur 530** : identifiants incorrects — réinitialiser le mot de passe FTP dans Manager.
