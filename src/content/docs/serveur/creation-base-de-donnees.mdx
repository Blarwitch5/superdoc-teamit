## Accès

1. Se connecter à [manager.infomaniak.com](https://manager.infomaniak.com).
2. Aller dans **Hébergement Web → [nom du service] → Bases de données**.

## Création

1. Cliquer sur **Créer une base de données**.
2. Renseigner :
   - **Nom** : identifiant de la base (ex. : `monsite_prod`)
   - **Mot de passe** : générer un mot de passe fort ou en saisir un (min. 12 caractères)
   - **Version MySQL** : choisir la version selon les prérequis de l'application (MySQL 8 recommandé)
3. Cliquer sur **Créer**.

## Informations de connexion

Après la création, noter les informations affichées :

| Paramètre | Valeur |
|-----------|--------|
| Hôte | `mysqliXXX.infomaniak.com` (fourni par Infomaniak) |
| Port | `3306` |
| Nom de la base | Le nom saisi à la création |
| Nom d'utilisateur | Identique au nom de la base (par défaut) |
| Mot de passe | Celui défini à la création |

Ces informations sont à renseigner dans le fichier de configuration de l'application (`.env`, `wp-config.php`, etc.).

## Accès à phpMyAdmin

1. Dans la liste des bases de données, cliquer sur l'icône **phpMyAdmin**.
2. Se connecter avec le nom d'utilisateur et le mot de passe de la base.

Depuis phpMyAdmin, il est possible d'importer un dump SQL, de créer des tables et d'exécuter des requêtes.

## Import d'un dump SQL

1. Dans phpMyAdmin, sélectionner la base de données.
2. Onglet **Importer** → choisir le fichier `.sql` ou `.sql.gz`.
3. Cliquer sur **Exécuter**.

Pour les fichiers volumineux (> 50 Mo), utiliser la commande MySQL en SSH si disponible :

```bash
mysql -h mysqliXXX.infomaniak.com -u utilisateur -p nom_de_la_base < dump.sql
```

## Sauvegardes

Infomaniak effectue des sauvegardes automatiques quotidiennes. Pour un export manuel depuis phpMyAdmin : onglet **Exporter → Rapide → SQL → Exécuter**.
