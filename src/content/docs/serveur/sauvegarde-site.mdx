## Vue d'ensemble

Une sauvegarde complète comprend deux éléments :
1. **Les fichiers** du site (code, médias, configuration)
2. **La base de données** (contenu, utilisateurs, paramètres)

À effectuer avant toute mise à jour majeure, migration ou intervention risquée.

## Sauvegarde des fichiers

### Via FileZilla (FTP/SFTP)

1. Se connecter au serveur (voir [Accès FTP/SFTP](/serveur/acces-ftp-sftp/)).
2. Naviguer jusqu'au dossier racine du site (généralement `/web/`).
3. Sélectionner tous les fichiers et dossiers (Ctrl+A).
4. Télécharger vers un dossier local dédié (ex. : `sauvegardes/monsite_AAAA-MM-JJ/`).

### Via le Manager Infomaniak

1. Se connecter à [manager.infomaniak.com](https://manager.infomaniak.com).
2. Aller dans **Hébergement Web → [nom du service] → Sauvegardes**.
3. Sélectionner un point de restauration ou déclencher une sauvegarde manuelle.
4. Télécharger l'archive générée.

## Sauvegarde de la base de données

### Via phpMyAdmin

1. Ouvrir phpMyAdmin depuis le Manager.
2. Sélectionner la base de données.
3. Onglet **Exporter**.
4. Choisir **Rapide**, format **SQL**.
5. Cliquer sur **Exécuter** — un fichier `.sql` est téléchargé.

### Via SSH (si disponible)

```bash
mysqldump -h mysqliXXX.infomaniak.com -u utilisateur -p nom_de_la_base > sauvegarde_AAAA-MM-JJ.sql
```

## Organisation des sauvegardes

Stocker les sauvegardes dans un emplacement séparé du serveur (NAS, cloud, poste local) avec la convention de nommage :

```
sauvegardes/
  monsite_2024-06-15/
    fichiers/
    base_de_donnees.sql
```

## Fréquence recommandée

| Type de site | Fréquence |
|-------------|-----------|
| Site vitrine (peu de mises à jour) | Mensuelle + avant chaque intervention |
| Site e-commerce / CMS actif | Hebdomadaire automatique + manuelle avant intervention |
| Application critique | Quotidienne automatique |

## Restauration

Pour restaurer :
1. **Fichiers** : uploader les fichiers via FTP en écrasant les existants.
2. **Base de données** : importer le dump `.sql` via phpMyAdmin (onglet **Importer**) ou via SSH.

Toujours tester la restauration dans un environnement de staging avant la production.
